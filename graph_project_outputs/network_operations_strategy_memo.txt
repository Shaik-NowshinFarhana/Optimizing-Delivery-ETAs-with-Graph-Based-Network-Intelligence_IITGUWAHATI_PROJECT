Project Title:
Optimizing Delivery ETAs with Graph-Based Network Intelligence

1. Executive Summary

The logistics network was modeled as a directed weighted graph where each facility acts as a node and each movement corridor acts as an edge. The edge weight was calculated using the median delay ratio between actual delivery time and OSRM-estimated delivery time.

This graph-based approach helps identify bottleneck hubs, delayed corridors, and structurally risky parts of the logistics network. It also improves ETA prediction by adding graph-level intelligence to the machine learning model.

4. Key Operational Findings

- Hubs with high betweenness centrality act as chokepoints because many routes pass through them.
- Corridors with delay ratio above 1.20 are chronic delay corridors.
- Graph-based features improve ETA prediction because they capture network risk.
- High-risk corridors should be reviewed for FTL movement instead of Carting.
- Bottleneck hubs contribute disproportionately to SLA breaches and revenue-at-risk.

5. Recommendations

Recommendation 1:
Upgrade processing capacity at the top 3 bottleneck hubs.

Recommendation 2:
Prioritize delayed corridors where actual time exceeds OSRM time by more than 20%.

Recommendation 3:
Use bottleneck score as a weekly operational monitoring metric.

Recommendation 4:
Shift high-risk long-distance corridors from Carting to FTL where delay risk is high.

Recommendation 5:
Use the graph-enhanced ETA model instead of relying only on OSRM estimates.
